# HopIn Docs

This docs folder is a simple learning guide for the project.

The docs are split by topic so it is easier to read one workflow at a time.

## Read this order if you are new

1. [backend-setup.md](./backend-setup.md)
2. [project-structure.md](./project-structure.md)
3. [database-schema.md](./database-schema.md)
4. [profile-page/README.md](./profile-page/README.md)
5. [home-find-rides/README.md](./home-find-rides/README.md)
6. [my-requests/README.md](./my-requests/README.md)
7. [my-rides/README.md](./my-rides/README.md)
8. [user-flow.md](./user-flow.md)

## What each doc is for

- `backend-setup.md`
  This explains how to connect Supabase and run the app.

- `project-structure.md`
  This explains the folder structure and MVC pattern in a simple way.

- `database-schema.md`
  This explains the main tables and how they connect.

- `profile-page/README.md`
  This explains the full Profile page flow from frontend to database.

- `home-find-rides/README.md`
  This explains the Home page, Find Ride page, filters, Leaflet map view, and ride details navigation.

- `my-requests/README.md`
  This explains rider view, driver view, booking requests, and request acceptance flow.

- `my-rides/README.md`
  This explains `Post a Ride`, `Request a Ride`, accepted rides, and role-based views.

- `user-flow.md`
  This is the bigger product-level summary of how the main workflows connect together.
